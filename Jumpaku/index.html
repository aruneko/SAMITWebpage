Use git!!

What's git?

gitとはヴァージョン管理システムです. ヴァージョン管理システムはファイル,フォルダの変更履歴を記録,追跡します. これを使うとソースコードの修正や統合を楽に行うことができます.

Start programming using git

gitを使った利用したプログラミングについて説明します. 準備するものは次の通りです.

  • GitHubのアカウント(https://github.com)
  • SourceTree(https://ja.atlassian.com/software/sourcetree/overview/)

以下は私がgitを使ってプログラミングを始める時の流れです.

  1. GitHubにリポジトリを作成する
  2. 自分のパソコンにcloneしてくる
  3. SourceTreeで初期化する
  4. .gitignoreを作成する
  5. 開発用のブランチを作成する

GitHubはフトウェア開発プロジェクトのための共有ウェブサービスです. 特に複数人で開発する場合にとても役に立ちます. リポジトリとはデータの貯蔵庫であり,フォルダのようなものです. GitHubを利用することでローカルのパソコン内ではなく,リモートのリポジトリを用意することができます. リポジトリを作成する時に"Initialize this repository with a README"チェックを付けてにREADME.mdを作成します. また"Add a lisence"でライセンスを指定できます.

GitHub上のリポジトリを自分のパソコンにコピーします. これがcloneです. SourceTreeの"新規/クローンを作成"からGitHubのリポジトリをクローンします.

SourceTreeの"Git Flow"からGit Flow用にリポジトリを初期化します.

.gitignoreはgit管理から外したいファイル,フォルダを書いておくファイルです. ここに書かれたファイル,フォルダは新しく作成されてもgit管理で管理されません. 既にgitで管理されているファイル,フォルダを無視する場合はgit管理から外してから.gitignoreに追加します. git管理から外すにはまずSourceTreeの"ターミナル"からコマンドプロンプトを起動し,コマンドを入力します. 例えば"Main.cpp"をgit管理から外すには git rm --cached Main.cpp とします. その後.gitignoreに"Main.cpp"と書き加えます.

初期化をするとmasterブランチとdevelopブランチができているはずです. developが開発用のブランチです. 現在のブランチをdevelopに移動し,ここから更に機能ごとにブランチを作成します. ブランチの作成にはSourceTreeの"Git Flow"から"新規フィーチャーを開始"を選択します.
